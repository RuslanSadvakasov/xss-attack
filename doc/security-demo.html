Демонстрация мер защиты

Ниже — наглядные примеры CSP, флагов cookie и безопасного вывода. Для реальных HTTP-заголовков запустите npm start (см. server.js).

Content-Security-Policy (CSP)

На этой странице в <meta> задан CSP: скрипты и стили только с вашего домена ('self'). Динамический inline-скрипт без nonce обычно блокируется (см. кнопку ниже и консоль DevTools). 

Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self'; ...

HttpOnly, Secure, SameSite

HttpOnly — JS не читает cookie (защита сессии при XSS).
Secure — только HTTPS. SameSite — снижает CSRF. 
Set-Cookie: session=abc; HttpOnly; Secure; SameSite=Strict; Path=/
